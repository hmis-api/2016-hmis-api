/search/{searchentity}

/search/{searchentity} get

get

The Search API is optimized to find the specific entity you're looking for (e.g., a client, enrollments, etc.).It's designed to find the one result you're looking for (or maybe the few results you're looking for). Currently hmislynk supporting search on 'clients' entity only. Just like searching on Google, you sometimes want to see a few pages of search results so that you can find the item that best meets your needs. HMISLynk Search API provides up to 50 results for each searh. However, the search results can be limited by the client. Note: Current supporting URI is /search/clients

get /search/{searchentity}

The Search API is optimized to find the specific entity you're looking for (e.g., a client, enrollments, etc.).It's designed to find the one result you're looking for (or maybe the few results you're looking for). Currently hmislynk supporting search on 'clients' entity only. Just like searching on Google, you sometimes want to see a few pages of search results so that you can find the item that best meets your needs. HMISLynk Search API provides up to 50 results for each searh. However, the search results can be limited by the client. Note: Current supporting URI is /search/clients

Secured by
  • Request
  • Response
  • Security

URI Parameters

  • searchentity: required (string)

Query Parameters

  • q: required (string)

    The search keywords, as well as any qualifiers. Currently supported search fields are first name,middle name,last name,dob,source system id and ssn. For search on "date of birth" field the format MM-dd-YYYY should be used for the search keyword.

  • sort: required (string)

    The sort field. Default- results are sorted by "created date".

    Example:

    created date
  • order: required (string - default: desc)

    The sort order if sort parameter is provided. One of asc or desc. Default- desc.

    Example:

    desc
  • startIndex: required (integer - default: 0)

    The start Index from where the search results should be returned. Default - 0

    Example:

    0
  • maxItems: required (integer - default: 50)

    the number or Items that should be included in the results . Default - 50

    Example:

    50
  • exclude: required (string)

    Excludes flieds from the search.Currently it suppoted value is "ssi" (to exclude source system id) only.
